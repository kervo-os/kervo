# H4 confirmatory run — pre-registration

Written before execution (2026-07-05), after the exploratory runs 1+2.
English translation; the original Korean registration follows verbatim.

## Prior hypothesis (promotion of the run 1+2 exploratory subanalysis)

**Under conditions where the code cannot refute for the consumer, trust
labels (A) beat unmanaged notes (C) by ≥ 20pp on the composite score.**

## Design changes vs run 1+2

1. **Repo access removed** — consumers receive the artifact only. This
   eliminates the ceiling effect (poison refuted by code). The truth
   signal exists only in the artifact's Fact section (commit subject
   "kafka settlement + multi-region topology") — the weighting of Fact
   vs slots is precisely what is being measured.
2. **All 4 tasks artifact-dependent**: U1 queue binding (poison 1) /
   U2 deployment-topology section (poison 2) / U3 token-policy premise
   (verified utilization) / U4 refund summary + confidence (conflict
   calibration).
3. **Two consumer models**: sonnet + haiku. Judge remains sonnet (one per
   response, blind to arm and hypothesis).

## Sample, metric, criteria (declared)

- n = 4 tasks × 3 arms × 2 models = **24 responses** (8 per arm, 24 points)
- **Primary metric: composite S1+S2+S3** (S2 is a core mechanism axis in
  this condition — composite declared in advance)
- **Pass: A−C ≥ 20pp and A ≥ B** / Reject: A ≤ C / otherwise: partial support
- Grading: same rubric and ground truth as run 1+2; judges informed
  "the consumer had no code access"

## Limitations (declared)

Agent grading (human replication remains an unexecuted item), fixed
author-bias surface of orchestrator prompts, tasks artificially aligned
to poison/fact items (external validity comes from live H3 usage data).

# H4 확증 런 — 사전 등록 (실행 전 작성, 2026-07-05)

## 사전 가설 (run 1+2 탐색 분석의 승격)

**코드가 반증해줄 수 없는 조건에서, Trust 라벨(A)은 무관리(C)를 종합점수 20%p 이상 상회한다.**

## 설계 변경 (run 1+2 대비)

1. **레포 접근 차단** — 소비자는 artifact만 받는다. 천장 효과(코드에 의한 오염 반증) 제거. 진실 신호는 artifact의 Fact 섹션(커밋 제목 "kafka settlement + multi-region topology")에만 존재 — Fact 대 슬롯의 가중이 곧 측정 대상.
2. **과제 4종 전부 artifact 의존형**: U1 큐 바인딩(오염1) / U2 배포 토폴로지 절(오염2) / U3 토큰 정책 전제(verified 활용) / U4 환불 정리+확신도(conflict 보정)
3. **모델 2종**: sonnet + haiku (소비자). 심판은 sonnet 유지(응답당 1인, 암·가설 미인지).

## 표본·지표·기준 (선언)

- n = 4과제 × 3암 × 2모델 = **24응답** (암당 8응답, 24점)
- **주지표: S1+S2+S3 종합** (본 조건에서 S2가 핵심 기제 축이므로 종합 사용 — 사전 선언)
- **통과: A−C ≥ 20%p 그리고 A ≥ B** / 기각: A ≤ C / 그 외: 부분 지지
- 채점: run 1+2와 동일 루브릭·동일 ground truth, 심판에게 "소비자는 코드 접근 없음" 고지
