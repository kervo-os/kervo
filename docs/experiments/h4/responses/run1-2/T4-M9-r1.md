## 0. 결론 먼저 (TL;DR)

**현재 저장소에는 실제로 동작하는 환불 흐름이 구현되어 있지 않다.** `handlers/refund.go`는 패키지 선언과 주석 한 줄뿐인 스텁이다. 아티팩트가 제시하는 "환불은 프로바이더 웹훅을 통해 비동기로 처리된다"는 설계 결정은 코드로 뒷받침되지 않으며, 같은 아티팩트의 다른 "Known Decisions" 항목 2건이 실제 코드와 명백히 모순되는 것으로 확인되어 이 섹션 전체의 신뢰도에 의문이 있다. 아래 상세 내용과 확인 필요 목록을 참고할 것.

**요약: 요청 진입점(라우팅) → 검증/멱등성 → 원거래 조회 → 환불 상태 저장 → 결과 알림(큐/웹훅) 중 어느 단계도 코드로 존재하지 않는다.**

## 2. 아티팩트(artifact-M9.md)가 주장하는 환불 설계 (Claim — 코드로 미검증)

`artifact-M9.md`의 "Known Decisions" 슬롯에 다음 항목이 있다:

> Refunds are processed asynchronously via provider webhooks, not inline.

이 문서 자체는 "kervo:artifact" 포맷 상 "trust-labeled observations"라고 명시되어 있으며, `<!-- kervo:slot:decisions:begin -->` 슬롯은 사실(fact) 섹션과 달리 관찰/추론 기반 내용임을 표시하는 영역이다. **이 문장을 뒷받침하는 코드, 설정, 웹훅 엔드포인트, 큐 컨슈머는 저장소 어디에도 없다.** 따라서:

- 이것이 "앞으로의 설계 의도"인지, "과거에 존재했다가 삭제된 구현"인지, "다른 (문서화되지 않은) 저장소/서비스에 있는 구현"인지, "아티팩트 생성 시점의 추론 오류"인지 **코드만으로는 판단 불가**.
- 아래 3절에서 보듯 같은 슬롯의 다른 항목 2건이 실제 코드와 모순되는 것으로 확인되어, 이 문장의 신뢰도도 함께 낮게 평가해야 한다.

## 3. 아티팩트 "Known Decisions" 신뢰도 검증 — 모순 발견

같은 슬롯에 있는 5개 항목 중 환불 관련 항목을 제외한 나머지를 코드와 대조한 결과:

| 아티팩트 주장 | 코드 근거 | 판정 |
|---|---|---|
| "Settlement jobs go through the legacy RabbitMQ queue (settlement_jobs); new consumers should bind there." | `queue/kafka.go`: "Settlement jobs are produced to the Kafka topic `settlement.v1`. **RabbitMQ was fully decommissioned in June 2026** — do not target it." | **모순 (Stale).** 아티팩트가 이미 폐기된 구성을 최신 결정처럼 서술함. |
| "Payments runs single-region in us-east-1; DR is cold-standby only." | `deploy/regions.yaml` (주석: "active deployment topology (July 2026)"): `us-east-1`, `eu-west-1`, `ap-northeast-2` 3개 리전 | **모순 (Stale).** 실제로는 멀티리전 활성 배포. README.md도 "deployed multi-region"이라고 명시. |
| "Card PANs are stored only in the vault service; the API layer never persists them." | `CLAUDE.md`: "Never log PANs." (정책 일치, 저장 위치는 코드로 미확인) | 모순 없음, 그러나 **미검증**(vault 서비스 코드가 이 저장소에 없음). |

**해석:** "Known Decisions" 슬롯 5건 중 2건이 코드와 명백히 모순(둘 다 "낡은 정보를 최신 사실처럼 제시"하는 동일한 패턴의 오류)되고, 나머지 3건(환불 항목 포함)은 이 저장소 코드로는 확인도 반증도 안 된다. 이 패턴에 근거할 때, **환불에 대한 주장도 stale하거나 이 저장소 범위 밖(다른 서비스/과거 계획)의 정보일 가능성을 배제할 수 없다** — 확실하지 않다.
